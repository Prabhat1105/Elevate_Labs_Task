Network Scanning Report

Objective

  • Discover active hosts within subnet
  • Identify open TCP ports and services
  • Evaluate potential risks

Note: IP addresses are partially hidden for security reasons.

Tools & Environment

ToolPurpose
NmapNetwork discovery and port scanning
WiresharkPacket capture & traffic analysis
OSWindows

Methodology

  1. Install Nmap from official website
  2. Identify local IP range (e.g., 192.168.1.0/24)
  3. Run nmap -sS 192.168.1.0/24 for TCP SYN scan
  4. Capture packets with Wireshark filter: tcp.flags.syn == 1 && tcp.flags.ack == 0

Analysis of Common Services

HTTP Proxy (Port 8080)

Forwards client requests to external servers for caching or filtering.

Oracle Database Listener (Port 1521)

Accepts client connections to Oracle databases.

Microsoft-DS / SMB (Port 445)

Used for file sharing and Active Directory communication in Windows environments.

Risks & Vulnerabilities

  • HTTP Proxy: Can be abused for spam, phishing, and traffic relaying.
  • Oracle Database Listener: Weak/default credentials and outdated versions pose security risks.
  • Microsoft-DS / SMB: Exploited by ransomware like WannaCry (EternalBlue/MS17-010).

Recommendations

HTTP Proxy: Require authentication, restrict to internal networks, monitor logs.

Oracle Listener: Disable default accounts, enable password authentication, restrict access by IP, apply patches regularly.

Microsoft-DS / SMB: Disable SMBv1, apply Windows patches, enforce Kerberos/NTLMv2, limit share permissions.

Close/disable unused ports and services, enforce strong passwords, enable MFA, keep firmware/software updated, and use firewalls.

Conclusion

The TCP SYN scan successfully identified active hosts and open services within the subnet. Applying the above security measures reduces attack surface and improves protection of network resources.
